Sendai - Vulnlab

Created2024-03-17|writeups

Sendai is a medium Active Directory machine from Vulnlab, created by xct. It involves resetting a user password with STATUS_PASSWORD_MUST_CHANGE status, abusing GenericALL permissions, and reading the gMSA password. For privilege escalation, we exploit ESC4 and ESC1.

NMAP

From the Nmap scan, we can see that we are dealing with a Domain Controller, as indicated by the presence of DNS, Kerberos and LDAP.

SMB

Let‚Äôs start by enumerating SMB using CrackMapExec

As shown, we can list shares with any user we enter and a null password. Let‚Äôs try to get a list of usernames using the --rid-brute flag

That worked, and we successfully got a list of all the users in the domain. Let‚Äôs get just the users and put them in a new file

We can attempt to spray a null password with this list of usernames

Password Change

We notice something interesting, the users Elliot and Thomas both have STATUS_PASSWORD_MUST_CHANGE. A quick google search led us to this post here that explains how to reset the password Resetting an Expired Password Remotely. Here, I set the password Passw0rd@ for the user Elliot.Yates. If it asks for the current SMB password we can just press enter for a blank password. Now that we successfully reset Elliot‚Äôs password, we can enumerate further

Elliot has read and write access to the config share, let‚Äôs use smbclient to connect and explore it

Great, we obtained the MSSQL credentials for the user sqlsvc. However, from the Nmap scan, we don‚Äôt see MSSQL

GenericALL

Let‚Äôs use bloodhound to get a better view of the network and what permissions our user has. I prefer to use BloodHound from CrackMapExec since it was integrated

As we can see, our user Elliot is a member of the Support group, which has GenericAll privileges to the OU ADMSVC@SENDAI.VL

We can simply add our user to the ADMSVC group

gMSA Password

Now that we are a member of the ADMSVC group, we can retrieve the password for the GMSA MGTSVC, since MGTSVC is a Group Managed Service Account.

Indeed, we were able to retrieve the NTLM of the mgtsvc$ account. Another tool we can use to get the gMSA password is gMSADumper

As observed, ADCS is being utilized on the machine. We can then use Certipy with the find command to identify templates

From the Certipy output, we can see that the SendaiComputer template is vulnerable to ESC4. We can refer to this HackTricks article Vulnerable Certificate Template Access Control - ESC4 or this YouTube video AD CS ESC4 Privilege Escalation Tutorial which explains how to abuse ESC4.

The first step is to make the certificate vulnerable to ESC1
